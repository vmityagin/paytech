# Адаптивная вёрстка — актуальные правила (2025)

---

## 1. РЕЗИНОВАЯ vs АДАПТИВНАЯ ВЁРСТКА

Два подхода — использовать оба вместе:

**Резиновая вёрстка** — элементы пропорционально масштабируются под размер окна,
не меняя своё расположение. Достигается через относительные единицы: `%`, `vh`, `vw`.

**Адаптивная вёрстка** — внешний вид элементов меняется по специальным правилам
при разных размерах окна. Реализуется через `@media` и `@container`.

Без JavaScript есть четыре способа реагировать на изменение размера окна:
1. Относительные единицы (`%`, `dvh`, `svh`, `vw`) — резиновость.
2. `@media` — CSS-правила для разных условий экрана.
3. `@container` — CSS-правила для разных условий контейнера.
4. `display: none` / `display: block` — показывать или скрывать элементы.

## 2. МЕТАТЕГ VIEWPORT — ОБЯЗАТЕЛЕН

Каждая HTML-страница ОБЯЗАНА содержать метатег viewport в `<head>`:
```html
<meta name="viewport" content="width=device-width, initial-scale=1">
```

- `width=device-width` — ширина страницы равна ширине экрана устройства.
- `initial-scale=1` — масштаб при загрузке равен 1, без увеличения.
- Без этого метатега мобильные браузеры отображают страницу как десктопную.

НЕ добавлять `maximum-scale=1` и `user-scalable=no` — это нарушает доступность
и запрещено стандартом WCAG 1.4.4. Пользователь должен иметь возможность
масштабировать страницу.

## 3. ОТНОСИТЕЛЬНЫЕ ЕДИНИЦЫ ИЗМЕРЕНИЯ

### Проценты `%`
- Размер задаётся относительно родителя.
- Для ширины работает надёжно. Для высоты — только если у родителя явно задана высота.
- `padding` и `margin` в процентах считаются от ШИРИНЫ родителя (включая вертикальные).
- Трюк для квадратного резинового блока (или использовать `aspect-ratio: 1`):
  ```css
  .block { width: 30%; aspect-ratio: 1; }
  ```

### Viewport-единицы
- `1vw` = 1% ширины окна. `1vh` = 1% высоты окна.
- `vmin` — меньшее из vw и vh. `vmax` — большее.
- `width: 100vw` создаёт горизонтальную прокрутку из-за полосы прокрутки — не использовать для основных блоков.

**Динамические viewport-единицы (использовать для мобильных интерфейсов):**
- `dvh` / `dvw` — динамические: учитывают появление/скрытие адресной строки браузера.
- `svh` / `svw` — наименьший viewport (адресная строка видна).
- `lvh` / `lvw` — наибольший viewport (адресная строка скрыта).

```css
/* НЕПРАВИЛЬНО для мобильного hero: 100vh обрезается из-за адресной строки iOS/Android */
.hero { height: 100vh; }

/* ПРАВИЛЬНО: dvh учитывает реальную высоту окна на мобильных */
.hero { height: 100dvh; }
```

Поддержка `dvh`/`svh`/`lvh` — все современные браузеры с 2023 года.

### min-width / max-width / min-height / max-height
- `max-width` на контентные зоны — стандартная практика для широких мониторов.
- `min-height` вместо фиксированного `height` для текстовых блоков.
- Избегать `max-height` без особой нужды — поведение непредсказуемо.

## 4. ФУНКЦИЯ calc()

Использовать `calc()` для вычисления размеров из комбинации единиц:
```css
.sidebar { width: 240px; }
.content { width: calc(100% - 240px - 32px); }

.column { width: calc(100% / 3); }
```

Правила синтаксиса `calc()`:
- Операторы `+` и `-` ОБЯЗАНЫ быть отделены пробелами: `calc(100% - 20px)`.
- Для `*` и `/` пробелы не обязательны, но рекомендуются.
- Можно комбинировать любые единицы: `calc(100dvh - 80px)`.

## 5. ЕДИНИЦЫ РАЗМЕРА ШРИФТОВ

Использовать `rem` для размеров шрифта. Это индустриальный стандарт доступности.

**Почему `rem`, а не `px`:**
- `px` — абсолютная единица. Если пользователь увеличил базовый шрифт браузера
  (настройки доступности, слабое зрение), сайт на `px` не масштабируется.
- `rem` привязан к базовому шрифту `<html>` (по умолчанию `1rem = 16px`).
  При масштабировании браузером всё увеличивается пропорционально.
- WCAG 1.4.4 требует, чтобы текст можно было увеличить до 200% без потери функциональности.

```css
/* Стандартная практика */
html { font-size: 16px; }   /* базовый размер для rem — глобальный сброс, исключение из правила классов */

.heading-1 { font-size: 2rem; }     /* 32px при дефолтных настройках */
.heading-2 { font-size: 1.5rem; }   /* 24px */
.text       { font-size: 1rem; }    /* 16px */
.text-small { font-size: 0.875rem; } /* 14px */
```

`em` — размер относительно шрифта родителя. Накапливается при вложенности,
использовать точечно (например, для отступов внутри компонента).

`line-height` задавать в относительных единицах — без единиц измерения:
```css
line-height: 1.5;   /* правильно */
line-height: 24px;  /* плохо — не масштабируется */
```

## 6. ИЗОБРАЖЕНИЯ

### Растровые изображения (jpg, png, webp)
- Использовать `srcset` для Retina-дисплеев вместо единственного `@2x`-файла:
  ```html
  <img
    src="photo.jpg"
    srcset="photo.jpg 1x, photo@2x.jpg 2x"
    alt="Описание"
    width="400"
    height="300"
  >
  ```
- Для адаптивных изображений разного размера использовать `sizes`:
  ```html
  <img
    src="photo-800.jpg"
    srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
    sizes="(max-width: 600px) 100vw, 50vw"
    alt="Описание"
  >
  ```
- Для смены кадрирования на разных экранах (art direction) — `<picture>`:
  ```html
  <picture>
    <source media="(max-width: 768px)" srcset="photo-portrait.jpg">
    <img src="photo-landscape.jpg" alt="Описание">
  </picture>
  ```
- Всегда указывать `width` и `height` атрибуты — браузер резервирует место
  до загрузки, предотвращает layout shift (CLS).
- Современный формат — `webp` (меньше на 25–35% при том же качестве).
  `jpg` — фолбэк.

### Загрузка изображений
```html
<!-- Изображение ниже первого экрана — ленивая загрузка -->
<img src="photo.jpg" alt="..." loading="lazy">

<!-- Главное изображение (LCP) — загружать приоритетно -->
<img src="hero.jpg" alt="..." fetchpriority="high">
```

### Векторные изображения (svg)
- Для логотипов и иконок — ВСЕГДА использовать `.svg`, не растр.
- SVG-файлы оптимизировать через SVGOMG: `https://jakearchibald.github.io/svgomg/`
- Способы добавить SVG (от лучшего к худшему):
  1. Inline SVG в HTML — управление через CSS/JS.
  2. `<img src="icon.svg" alt="...">` — просто, без CSS/JS доступа.
  3. `background-image` в CSS — для декоративных элементов.
  4. `<iframe>`, `<embed>`, `<object>` — не использовать.

### Общие правила
- `alt` — обязателен для всех `<img>`. Для декоративных: `alt=""`.
- `max-width: 100%` для резиновых изображений.
- `aspect-ratio` для сохранения пропорций без padding-трюков:
  ```css
  .image-wrapper { aspect-ratio: 16 / 9; overflow: hidden; }
  ```

## 7. ОПТИМИЗАЦИЯ ШРИФТОВ

- Использовать формат `.woff2` — поддерживается всеми современными браузерами,
  на 30% легче `.woff`. Подключать `.woff` только как фолбэк для устаревших браузеров.
  ```css
  @font-face {
    font-family: 'MyFont';
    src:
      url('myfont.woff2') format('woff2'),
      url('myfont.woff') format('woff');
    font-display: swap;
  }
  ```
- `font-display: swap` — текст отображается немедленно системным шрифтом,
  заменяется кастомным после загрузки. Улучшает LCP.

- Сглаживание шрифтов — применять на корневом классе страницы (не на теге `body`):
  ```css
  .page {
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
  }
  ```

- `text-rendering: optimizeLegibility` замедляет рендеринг —
  применять точечно только в заголовках, не глобально.

## 8. МЕДИАЗАПРОСЫ И КОНТЕЙНЕРНЫЕ ЗАПРОСЫ

### @media — для условий экрана
```css
@media screen and (max-width: 768px) {
  .header { font-size: 1.25rem; }
}
```

### @container — для условий контейнера (предпочтительно для компонентов)
Контейнерные запросы позволяют компоненту адаптироваться к размеру своего
родителя, а не всего экрана. Это фундаментально важно для переиспользуемых
компонентов.

```css
/* 1. Объявить контейнер */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

/* 2. Написать стили для компонента внутри */
@container card (max-width: 400px) {
  .card { flex-direction: column; }
}

@container card (min-width: 401px) {
  .card { flex-direction: row; }
}
```

Поддержка `@container` — все современные браузеры с 2023 года.

Правило выбора: если поведение зависит от ширины всей страницы — `@media`.
Если от размера конкретного блока-обёртки — `@container`.

### Стандартные брейкпоинты
```css
/* Mobile First — базовый код для мобильных */

@media screen and (min-width: 375px) { /* средний смартфон */ }
@media screen and (min-width: 425px) { /* широкий смартфон */ }
@media screen and (min-width: 768px) { /* планшет */ }
@media screen and (min-width: 1024px) { /* планшет альбом / ноутбук */ }
@media screen and (min-width: 1440px) { /* десктоп */ }
@media screen and (min-width: 2560px) { /* 4K */ }
```

Подход Mobile First (`min-width`) — предпочтительный для продуктов
с мобильным приоритетом трафика (большинство современных сайтов).
Desktop First (`max-width`) — допустим, если макет начинается с десктопа.

Контентные брейкпоинты — ставить медиазапрос там, где макет начинает
ломаться, а не только по стандартным значениям устройств.

### Размещение медиазапросов
- В монолитном CSS — медиазапросы в конце файла.
- В компонентном подходе (БЭМ, CSS Modules) — медиазапрос каждого
  компонента в конце его CSS-файла.

### Анализ макета перед вёрсткой
Перед написанием медиазапросов ответить на 5 вопросов:
1. Как меняется количество строк и колонок при сжатии?
2. Меняют ли элементы порядок относительно друг друга?
3. Как меняются размеры и отступы — найти закономерности.
4. Какие элементы пропадают и появляются на разных разрешениях?
5. Как изменение элементов связано с изменением их контейнеров?

## 9. СТРОГИЕ ПРАВИЛА АДАПТИВНОСТИ

1. Метатег `viewport` с `width=device-width, initial-scale=1` — ОБЯЗАТЕЛЕН.
   `maximum-scale` и `user-scalable=no` — ЗАПРЕЩЕНЫ (нарушение WCAG).
2. Никаких фиксированных `px` для ширины блоков — только `%`, `max-width`, `calc()`.
3. Высота блоков — через `min-height`, не фиксированный `height`.
4. Высота на мобильных — `dvh` вместо `vh` (адресная строка браузера).
5. Размер шрифта — `rem`, не `px`. `line-height` — без единиц (1.2 или 1.5).
6. Изображения — `srcset` + `sizes` для растра; только SVG для иконок/логотипов.
7. Всегда указывать `width` и `height` атрибуты у `<img>`.
8. LCP-изображение — `fetchpriority="high"`. Изображения ниже экрана — `loading="lazy"`.
9. Шрифты — `.woff2` с `font-display: swap`. `.woff` — только фолбэк.
10. Медиазапросы — в конце CSS-файла.
11. Контейнерные запросы `@container` — для компонентов, зависящих от родителя.
12. В `calc()` операторы `+` и `-` — всегда с пробелами.
13. `width: 100vw` — не использовать для основных блоков.
14. `text-rendering: optimizeLegibility` — только точечно в заголовках.
15. SVG — оптимизировать через SVGOMG перед использованием.
16. `aspect-ratio` — для пропорциональных блоков вместо `padding-bottom` трюка.
